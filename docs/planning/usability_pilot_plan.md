# Usability Pilot Plan

## 0. Purpose

This document defines a small pilot usability study for the Smart Image Part Identifier prototype.

The goal is not to claim statistically strong human-subject evidence, but to collect defensible early evidence about:

- interface clarity,
- shortlist usefulness,
- evidence usefulness,
- metadata-preview usefulness,
- perceived trust and effort reduction.

## 1. Recommended Study Size

- Minimum acceptable: 3 participants
- Preferred: 5 to 8 participants
- Ideal mix:
  - users with secondhand marketplace experience,
  - at least one user who has listed technical or electronic items,
  - optionally one domain-familiar operator

## 2. Study Setup

Each participant performs three short tasks using the prototype.

Recommended setup:

- one facilitator,
- one participant at a time,
- screen recording if permitted,
- stopwatch or automatic timing capture,
- post-task Google Form.

## 3. Task Set

### Task 1. Identify from image

The participant is given one product image and asked to use the prototype to determine:

- likely maker,
- likely part number,
- likely product type.

### Task 2. Judge shortlist usefulness

The participant is given an image or query and asked to inspect the Top-K results and decide which candidate looks most plausible.

### Task 3. Review and refine metadata

The participant uploads an image, inspects the generated metadata draft, edits fields if necessary, and states whether the draft is usable for listing preparation.

## 4. Quantitative Measures

For each task, record:

- task completion time,
- whether the participant reached a plausible final answer,
- number of manual metadata edits,
- whether external search was needed,
- whether the correct item appeared in the shortlist if known.

## 5. Post-Task Questionnaire

After the tasks, participants complete a short Google Form.

Recommended Likert items:

1. I could understand how to use the interface without help.
2. The search results were useful for narrowing down candidate parts.
3. The evidence shown with the results helped me judge whether the result was trustworthy.
4. The metadata preview reduced the effort needed to prepare listing information.
5. I would prefer using this prototype over completely manual search for similar tasks.
6. I felt confident making a decision from the shortlist provided by the system.

Open-ended questions:

- What was the most useful part of the prototype?
- What was the most confusing or frustrating part?
- What should be improved first?

Background questions:

- Have you used secondhand marketplaces before?
- Have you ever listed technical, industrial, or electronic items before?

## 6. Reporting Guidance

If the study is not completed before submission, the report should say:

> A small pilot usability study has been designed using three listing-oriented tasks and a short post-task Google Form, but aggregate results are still pending.

If the study is completed, the report should say:

> A small pilot usability study was conducted with N participants. The study recorded task completion time, correctness, edit count, and self-reported usability/trust ratings collected through a Google Form.

## 7. Minimum Defensible Output

Even a minimal pilot should report:

- number of participants,
- task definitions,
- average task time,
- main usability positives,
- main usability pain points.
