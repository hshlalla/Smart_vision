# Submission Guide

`submission/`은 제출 관련 문서를 보관하는 폴더다.  
원칙은 다음과 같다.

- 실제 제출한 보고서/PDF/DOCX는 `reports/`
- 제출 후 받은 피드백은 `feedback/`
- 가이드, rubric, webinar 자료는 `guides/`
- 제출용 일정 자료는 `planning/`
- 테스트/재현 증적은 `evidence/`

## Structure

- `reports/`
  - `preliminary.pdf`
  - `Draft.docx`
  - `Draft.pdf`
- `feedback/`
  - `preliminary_feedback.md`
  - `draft_feedback.md`
- `guides/`
  - `final_report_guide.md`
  - `final_report_rubric_v2 (1).pdf`
  - `fp-preliminary-rubric.pdf`
  - `Webinar Notes.pdf`
- `planning/`
  - `gantt_chart.xlsx`
- `evidence/`
  - `report_support_2026-03-10/`

## Canonical Final Report Workflow

최종보고서는 아래 흐름으로 정리한다.

1. `reports/Draft.docx`를 2차 제출본 기준선으로 사용
2. `feedback/draft_feedback.md`와 `feedback/preliminary_feedback.md`를 반영
3. `guides/final_report_guide.md`와 rubric에 맞춰 장 구조를 최종 조정
4. `evidence/report_support_2026-03-10/`의 테스트/환경/변경 요약을 근거로 Evaluation/Implementation 내용을 보강
5. 작업 중인 상태 정리는 `docs/reports/final_report_status.md`에서 추적
